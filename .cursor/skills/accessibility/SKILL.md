# Accessibility (a11y)

## Scope

- Semantic HTML, ARIA and roles
- Keyboard navigation and focus
- Contrast, sizes and readability
- Screen readers and testing (axe, Lighthouse)

## When documenting context

- Indicate if the project has a11y requirements or standard (WCAG)
- Reference accessible components in the design system
- Mention testing tools used

## When generating rules

- Include rules for labels, alt text and focus in components
- Reference WCAG or the project's a11y guide
