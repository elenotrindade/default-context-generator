# Testing

## Scope

- Unit, integration and e2e tests
- Mocks, fixtures and test data
- Coverage, reports and CI integration
- Naming patterns (describe, it, test) and folder structure

## When documenting context

- Indicate test stack (Jest, Vitest, pytest, Cypress, etc.)
- Describe where tests live (next to code vs. separate folder) and conventions
- Mention how to run tests and what coverage is expected

## When generating rules

- Include conventions for writing tests and using mocks
- Reference the project's test framework docs
