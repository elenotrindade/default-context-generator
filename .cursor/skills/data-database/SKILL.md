# Data / Database

## Scope

- Modeling (schemas, entities, relationships)
- Migrations, seeds and schema versioning
- Queries, indexes and performance
- ETL, caches and storage (relational, NoSQL, queues)

## When documenting context

- Describe database(s) used, ORM or query builder
- Indicate where migrations and naming conventions are
- Mention backup policy, data environment and sensitivity

## When generating rules

- Include rules for safe migrations and avoiding sensitive data in logs
- Reference database and project ORM docs
