# UX/UI

## Scope

- User flows, journeys and wireframes
- Design system (tokens, visual components), visual accessibility
- Microcopy, labels and interface messages
- Consistency of patterns (forms, feedback, navigation)

## When documenting context

- Describe the project's design system or style guide
- Indicate tools (Figma, Storybook, etc.) and where to find references
- Mention accessibility conventions adopted

## When generating rules

- Include references to design system and component guide
- Suggest consulting a11y docs when relevant
