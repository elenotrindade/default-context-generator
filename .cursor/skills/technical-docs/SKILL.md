# Technical Documentation

## Scope

- README, CONTRIBUTING, architecture and ADRs
- API documentation (OpenAPI, comments)
- Setup, deploy and troubleshooting guides
- Code comments and JSDoc/docstrings

## When documenting context

- Indicate where the main documentation is and how it is kept up to date
- List tools (Swagger, Docusaurus, etc.) and format (Markdown, etc.)
- Mention writing conventions (tone, structure, examples)

## When generating rules

- Include rule to keep README and docs in sync with code
- Reference project doc style (e.g. JSDoc, OpenAPI)
