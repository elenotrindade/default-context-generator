# Area: Extension (Cursor/VS Code)

## What this area does in the project

The extension exposes a **status bar button** (bottom right: "Generate context") and **commands** in the palette. When you click the button:

1. It opens a **popup (webview)** with:
   - **Repository language** — dropdown to choose the language for generated docs and rules: **English**, **Portuguese**, or **Mixed**. This choice is used when you click "Generate context" so the output is in the selected language.
   - **Generate context** — runs the generation flow (uses the prompt from the config file if it exists, or the default prompt).
   - **Set or create config file** — lets you create a new file (with the default prompt) or point to an existing one; the path is stored in `defaultContextGenerator.configPath` (workspace/global).
2. If no config file is set yet, the popup shows the **default prompt** so the user can create/customize it in one flow.
3. When clicking "Generate context": it checks for an open workspace, runs the **Cursor CLI** (`agent -p --force "<prompt>"`), and output goes to the **"Default Context Generator"** panel with an **"Environment ready!"** notification when done. When running from the **command palette** (without the popup), if the setting `defaultContextGenerator.repoLanguage` is **"Ask every time"**, a language picker is shown; otherwise the configured language is used.

Requires the [Cursor CLI](https://cursor.com/docs/cli/installation) installed and on PATH.

**Auto mode:** to avoid unnecessary client usage, the extension can run context generation **automatically** when a workspace is opened and has no context yet (`docs/context/` or `.cursor/rules/` missing or empty). Settings: `defaultContextGenerator.autoGenerate` (default true), `defaultContextGenerator.autoGenerateOnlyWhenMissing` (default true), `defaultContextGenerator.autoGenerateDefaultLanguage` (en/pt/mixed), `defaultContextGenerator.model` (agent model for auto-run). Auto runs at most once per workspace (state in globalState) unless the user runs the command manually.

**Model selector:** when generating from the command palette, a model picker is shown (Default, Claude 4.6 Opus, Claude 4.6 Sonnet, Claude 4.5 Sonnet, Claude 4 Haiku, or the value from `defaultContextGenerator.model`). This avoids always using the default (e.g. Opus) and lets you choose a lighter model to reduce usage.

**Repository language:** `defaultContextGenerator.repoLanguage` (en / pt / mixed / ask). When **ask** (default), the command palette run shows a language picker. When set to en, pt, or mixed, that language is used directly. The popup always shows a dropdown so you can choose the language before generating.

## Where it is in the code

- **`package.json`** — manifest: `main`, `contributes.commands`, `activationEvents`.
- **`src/extension.ts`** — extension logic:
  - `PROMPT_DEFAULT`: default prompt; used when there is no config file.
  - Config: `defaultContextGenerator.configPath` (setting); file can be `.txt` (text = prompt) or `.json` with a `"prompt"` field. Auto: `autoGenerate`, `autoGenerateOnlyWhenMissing`, `autoGenerateDefaultLanguage`, `model`. Model selector uses `--model` when running the agent. Repository language: `repoLanguage` (en/pt/mixed/ask).
  - Popup (webview) with repository language dropdown, "Generate context", and "Set or create config" buttons; when no config is set, it shows the default prompt.
  - `activate()`: registers commands and status bar; schedules `tryAutoRun()` after 4s when workspace has no context (if auto is enabled).
- **`out/extension.js`** — build (from `npm run compile`).
- **`.vscode/launch.json`** — config for F5 (Extension Development Host).

## Conventions and decisions

- The prompt is **self-contained**: it describes the workflow steps for the agent to follow even without the skill file in the workspace.
- Chat-opening commands are tried in order: `composer.newAgentChat`, `aichat.openChat`, `cursor.chat.open`, `workbench.action.chat.open`.
- No runtime dependencies beyond VS Code/Cursor; only TypeScript and `@types/vscode` in dev.
