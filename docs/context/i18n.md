# Area: Internationalization (i18n)

## What this area does in the project

Keeps **English** as the primary language for code, rules, skills, and `docs/context/`, with **Portuguese** as the secondary language for contributors and users who prefer it. The **VS Code/Cursor extension UI** follows the editor display language using NLS (national language support) files and runtime message maps.

## Where it is in the code

- **Rule:** [.cursor/rules/i18n.mdc](../../.cursor/rules/i18n.mdc) — conventions for en/pt content.
- **Manifest strings:** `package.nls.json` (English), `package.nls.pt.json` (Portuguese) — command titles, configuration labels.
- **Runtime UI:** `src/nls.ts` — `messages.en` / `messages.pt` for popups, status bar, and notifications.
- **User-facing docs:** `README.md` (primary), `README.pt.md` (Portuguese).

**Skill folder names** (e.g. `technical-docs`, `software-architecture`) stay in **English** everywhere so Cursor can resolve `.cursor/skills/<name>/SKILL.md` consistently, including in Portuguese documentation.

## Conventions

- When adding commands or settings: update both `package.nls.json` and `package.nls.pt.json`; keep keys aligned.
- When adding runtime strings: add entries to both `messages.en` and `messages.pt` in `src/nls.ts`.
- Default **agent prompt** text embedded in the extension remains **English** for consistent agent behavior across locales.
